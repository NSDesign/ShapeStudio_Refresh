# Appendix A — Shape System & Canvas

Detail for the shape primitives, geometry, boolean operations, grouping, and the canvas viewport. Implemented and verified in code.

---

## A.1 Shape types (31)

A single `SupportedShapeType` union enumerates every primitive. All are generated by the same pipeline; each may carry type-specific properties (A.4).

```
rectangle | rounded-rectangle | square | rounded-square |
circle | ellipse | semicircle | ring |
triangle | right-triangle | trapezoid | pentagon | hexagon |
rhombus | parallelogram | kite |
heart | arrow | cross |
line | line-vector |
polygon | star | chunk | blob |
cubic | bezier | smooth-spline | spline-circle | spline-ellipse | spline-ring
```

Notes that matter for a faithful rebuild:
- **Polygon** has a configurable segment/edge count; **star** has inner-radius control; **ring** and the spline-ring/circle/ellipse variants have inner-radius / segment controls.
- **Curve families** (`cubic`, `bezier`, `smooth-spline`) are point-based with control points, open/closed probability, and stroke caps; splines have point-count, point-position, and control-point modes.
- **`chunk`** and **`blob`** are organic/irregular generated forms.
- `line` vs `line-vector` are distinct (a rendered line primitive vs a vector line).

## A.2 Shape model & geometry

Each shape is an instance of a `Shape` class (`client/src/lib/shapes.ts`) holding:
- Identity & ordering (`id`, z-index/layer).
- Type and the generated **points** that define its geometry.
- Position, size, rotation, scale, skew (transform state).
- Visual properties: fill (solid/gradient), stroke, opacity, blur, shadow/glow (Appendix D), blend mode, compositing operation.

Critical behaviour: whenever size/shape-defining parameters change, the shape must **regenerate its points** (`regenerateShapePoints()`). This is a known correctness dependency — size application must call regeneration before rendering, or geometry desyncs from properties. A rebuild must preserve this "mutate params → regenerate points → render" ordering.

Transforms are applied relative to a configurable **transform origin** (center / corners / shape-reference / artboard-anchor), in the order: scale → rotate → translate (+ skew, + additive randomisation). See Appendix B §B.6.

## A.3 Boolean operations

`BooleanOperations` (`client/src/lib/booleanOperations.ts`) implements four operations on shapes:

```
union | subtract | intersect | exclude
```

Implementation detail to reproduce: operations are computed via canvas global-composite-operation compositing plus point-level geometry, with guards — `subtract`/`intersect` return null when the result has fewer than 3 points (nothing left / no overlap), and a warning path when shapes don't intersect. Boolean ops are invoked from `useShapeEditor` (user selects shapes → applies operation → result replaces inputs).

## A.4 Type-specific properties

Per-shape-type parameters live in `ShapeSpecificProperties` (and corresponding fields on `BatchConfigSettings`). Each is expressed with the standard mode pattern (`range` / `define` / `incremental`). Examples that must exist:
- Rectangle family: corner-radius mode.
- Star: inner-radius mode.
- Ring / spline-ring: inner-radius mode.
- Polygon: segment/edge-count mode.
- Lines: point-count, point-position modes.
- Splines/curves: spline point-count, point-position, control-point modes; open probability; stroke-cap probabilities; curvature/spread (for `cubic`).

## A.5 Grouping

Hierarchical grouping via a `ShapeGroupClass`. Groups can be selected, transformed, and composited as units. Selection state tracks both `selectedShapes` and `selectedGroups` independently.

## A.6 Canvas viewport

`Canvas.tsx` implements an **infinite, layered canvas**:
- Three-layer rendering pipeline (background / content / overlay-adornments).
- Pan & zoom (mouse + multi-touch), with the artboard rendered within world space.
- Selection: click, additive (shift/ctrl), and marquee; supports multi-selection of shapes and groups.
- Real-time re-render on state change, optimised to avoid full redraws where possible.
- Adornments/overlays (grid, artboard geometry, selection handles) are a separate layer and are independently includable in exports.

## A.7 Artboard

- Artboard presets (`client/src/lib/artboardPresets.ts`) + custom dimensions; background colour; DPI.
- `SavedArtboard` is persisted in user preferences (multiple saved artboards, with auto-select of newly created artboards and safe deletion behaviour).
- Print-oriented artboard config (bleed, safe zone, marks, units) is detailed in Appendix E.
